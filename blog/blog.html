closures

February 17, 2018

One of the biggest questions that got me stuck in my most recent phone interview was closure. Not the relationship type of closure, but JavaScript closure. I told the interviewer in an act of nervousness that closurse were "inner functions that have access to global variables." That wasn't very... correct. Not at all. In short from what I've found online, closures are functions with preserved data. Let me explain after this code block.


var addTo = function(passed) {
    var add = function(inner) {
        return passed + inner;
    };
    return add;
};

var addThree = new addTo(3);
var addFour = new addTo(4);

console.dir(addThree); // Closure is 3
console.dir(addFour); // Closure is 4

// Closure preserves a value inside of the function
// Closures are functions with preserved data

console.log(addThree(1)); // Output is 4
console.log(addFour(1)); // Output is 5

Click here to see code on JSFiddle

To be demonstrate how it all works, I have the above example. We have a function called addTo that takes in a parameter passed. Then, inside we have an inner function named add that takes in another parameter inner. addTo returns add, and add returns passed + inner. So the entire function relies on add to finish to return the addition of the two numbers.

Next, I declare two variables that create a new addTo object that takes in 3, and the other taking in 4 as their parameter. Now if you console.dir instead of console.log, it'll give you a detailed look of the objects. In the Console of Chrome's Developer tools you will see a f add(inner) with a little dropdown arrow next to it. After clicking the dropdown, and you go to [[Scopes]] you'll see a closure dropdown that will show a "3" for your first variable and a "4" for the other. This is demonstrated below.

So, the variables we made will always preserve these closures. No matter how many times we console.log(addThree(1)), addThree will always preserve that variable 3. The variable 3 is now a private variable to the object.

w3schools defines closures pretty well.

A closure is a function having access to the parent scope, even after the parent function has closed.

Even though the function is closed, our variables are still holding onto these private variables even though they cannot be directly called, unless you invoke the second inner function.
